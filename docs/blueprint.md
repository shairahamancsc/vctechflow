# **App Name**: ServiceFlow

## Core Features:

- Service Request Portal (Customer): Customers can submit service requests for their printers, including details about the issue and printer model.
- Technician Task Management: Technicians receive notifications for new service requests, manage pickup and delivery schedules, and update service status.
- Status Tracking (Customer): Customers can track the real-time status of their service request through the portal.
- Service Log and Reporting: Technicians can log the details of the service performed, including parts replaced and labor time.
- Automated Notification System: The system sends automated notifications to customers and technicians at each stage of the service process (request received, technician dispatched, printer ready for delivery, printer delivered).
- Parts Inventory: A record of available replacement parts.

## Style Guidelines:

- Primary color: Light blue (#5F9EA0) for reliability and trust.
- Background color: Very light blue (#E0FFFF) to keep the UI light and airy.
- Accent color: Soft green (#8FBC8F) for 'available' or completion indicators, and calls to action.
- Body and headline font: 'PT Sans' for a balance of modernity and readability.
- Simple, clear icons to represent service statuses and actions.
- Clean and intuitive layout with clear distinctions between customer and technician portals.
- Subtle transitions and loading animations to enhance user experience.
